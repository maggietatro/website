CSS VARIABLES

CSS Variables are hard to describe without sounding scarily technical, however they are pretty easy to implement. Imagine defining a color in one place and then referencing that color throughout your css. If you need to change the color, you just change it once and all the references update automatically. It is kind of like setting a Character Style in InDesign or a CSS value in HTML (I know that's pretty meta).

  • getting started with css variables explains the system quite well
  • w3schools explanation of css variables
  • creative bloq calls them css custom properties
  • mozilla developer network also calls them css custom properties

CSS TRANSFORMS & TRANSITIONS

We are getting into animation and more interactive effects. As a lead-in to actual animation, I would like to cover transforms and transitionsin CSS.

Transforms are a way to use CSS to change placement, size, rotation, and other visual settings. These can be used to change an element's default look, but are also good for creating dynamic hover effects. We will also use transforms quite a bit in our actual animations as well.

  • w3schools transform breakdown is a good starting reference
  • w3schools explanation of 2d transforms
  • w3schools explanation of 3d transforms
  • mozilla developer network has a good breakdown too
  • sitepoint has some great practical uses for transforms that aren't just about hover effects

Transitions are a way to add timing to an element's visual changes. Slowing down a change when hovering over an element is a good use for transitions. While not exactly animation, some of the concepts are definitely relevant.

  • w3schools explains the basics of transitions
  • mozilla developer network explains the basics of transitions
  • mozilla developer network has a few examples of transitions in action
  • css-playground has an interactive tool to let you explore transition hover effects
